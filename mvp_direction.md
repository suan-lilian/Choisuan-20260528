[MVP 정의]

핵심 기능 우선순위

1. 사용자 clone Agent 형성
2. 장바구니 자동 구성 → Journey Map 최대 병목(30분) 개선
3. 수정 인터페이스 → AI 신뢰 낮은 초기 단계에서 사용자 이탈 방지

===========================
핵심 기능 1. AI Agent 형성
===========================

User Edge (사용자 구매 패턴·성향 데이터)

- 구매 이력 (품목, 수량, 주기)
- 구매 시간대 (퇴근 후, 주말 등)
- 가격 민감도 (얼마까지 허용하는가)
- 우선순위 설정 (가격 vs 신선도 vs 브랜드)
- 수정 이력 (AI 추천을 얼마나, 어떻게 바꿨는가)
- 가족 구성 (인원, 연령대)
- 기피 식품 / 알레르기
- 즐겨 만드는 메뉴 패턴

Product Edge (식자재 데이터)

- 가격 (현재가, 할인율, 타 플랫폼 비교가)
- 리뷰 (신선도·품질 관련 키워드 분석)
- 원산지 / 성분 정보
- 유통기한 / 입고 시기
- 재고 여부
- 묶음 단위 / 중량 대비 단가
- 브랜드 신뢰도
- 계절성 (제철 식재료 여부)

===========================
핵심 기능 2. 사전 장바구니 자동 구성 & 선택 근거 명시
===========================

- 사용자가 앱을 열기 전, AI가 예상 구매 목록을 미리 구성해 장바구니에 담아둠
- AI가 각 상품을 고른 이유를 사용자에게 투명하게 보여줌
  ex) "이 계란을 고른 이유: 지난 3회 구매 이력 + 현재 최저가 + 리뷰 내 신선도 언급 87%"
  → 탐색~비교 구간(30분) 단축으로 병목 제거

===========================
핵심 기능 3. 수정 인터페이스
===========================

AI 선택이 마음에 안 들 때 수정 요청 가능

수정 유형 | 예시
전체 재구성 | "냉면 먹을거라서 다시 짜줘"
개별 상품 교체 | "계란 다른 걸로 바꿔줘"
기준 변경 | "이번엔 가격보다 신선도 우선해"

→ AI를 완전히 신뢰하기 어려운 초기 단계에서 사용자 통제권 확보
